Destacado

Landing Page - Contabilidad Florez

Proyecto real de principio a fin para un despacho contable: landing page responsive con marca propia, dominio, DNS y correo corporativo configurados desde cero, dejando el sitio en producción, con HTTPS e indexado en Google.

  • HTML5
  • CSS3
  • Cloudflare_DNS
  • SEO_Técnico

Más que una landing page: un proyecto completo de presencia digital para un cliente real. Además del frontend (HTML5, CSS3 y JavaScript, responsive y con validaciones), se gestionó la compra del dominio propio, la zona DNS en Cloudflare (registros A y CNAME), el despliegue en GitHub Pages con dominio personalizado y HTTPS, el correo corporativo mediante Cloudflare Email Routing con SPF y DMARC, y el SEO técnico (metaetiquetas, Open Graph, datos estructurados Schema.org e indexación en Google Search Console).

Lo que aprendí:

  • A llevar un sitio de principio a fin como lo haría una agencia: branding, desarrollo, infraestructura y servicios empresariales, no solo el código.
  • A entender y configurar la relación entre dominio, DNS y hosting: compra del dominio, delegación a Cloudflare y publicación con dominio propio en GitHub Pages.
  • A administrar registros DNS (A, CNAME, MX, TXT) y a habilitar HTTPS mediante el proxy/CDN de Cloudflare.
  • A montar correo corporativo sin servidor propio usando Cloudflare Email Routing, y a configurar SPF y DMARC para reducir el riesgo de suplantación.
  • A aplicar SEO técnico real (metaetiquetas, Open Graph, datos estructurados Schema.org) y a indexar el sitio en Google Search Console.
  • A integrar servicios de terceros como Formspree y WhatsApp para resolver funcionalidad de backend sin salir de un sitio estático.

Formulario de registro con Validaciones

Formulario de registro que reúne prácticamente todos los tipos de input de HTML5, usado como ejercicio para dominar las validaciones nativas del navegador y construir formularios accesibles sin depender de JavaScript ni librerías externas.

  • HTML5
  • CSS3

Ejercicio enfocado en explotar al máximo las capacidades nativas de HTML5 para formularios: más de 20 campos distintos (texto, email, rango, color, fecha, semana, mes, archivos, entre otros) con validaciones mediante required, pattern, minlength y datalist para sugerencias. Se prestó especial atención a la accesibilidad, usando fieldset, legend y etiquetas asociadas a cada campo, además de un diseño con etiquetas flotantes.

Lo que aprendí:

  • Implementación de validaciones nativas como required, pattern, min y max, aprovechando el comportamiento del navegador.
  • Uso de fieldset y legend para estructurar formularios de forma semántica y accesible.
  • Estilización de formularios manteniendo consistencia entre inputs, selects y botones, incluyendo estados como :focus y :valid/:invalid.
  • A usar datalist para ofrecer sugerencias de autocompletado sin necesidad de JavaScript.

Contenido Multimedia y Responsive

Página informativa sobre contenido multimedia responsive: imágenes adaptativas, video y audio en HTML5. Documenta con ejemplos funcionales las técnicas para optimizar el peso y la calidad visual del contenido según el dispositivo del usuario.

  • HTML5
  • CSS3

Proyecto pensado como referencia técnica sobre multimedia responsive. Además de los ejemplos funcionales (srcset, picture, video y audio con múltiples formatos, video embebido de YouTube), incluye tablas comparativas con formatos, pesos y resoluciones recomendadas para imágenes, video y audio, a modo de guía de buenas prácticas aplicables a futuros proyectos.

Lo que aprendí:

  • Uso de srcset y sizes para servir imágenes adaptadas según el dispositivo.
  • Implementación del elemento picture para control de versiones de imagen (art direction).
  • A elegir el formato de audio/video adecuado según el caso de uso y a diferir su carga con preload para no afectar el rendimiento.
  • A investigar y documentar pesos y resoluciones recomendadas por tipo de contenido, como criterio objetivo para futuras optimizaciones.
